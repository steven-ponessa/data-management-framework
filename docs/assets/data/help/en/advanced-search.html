In the basic search, users can filter through node values and select nodes that meet the filter criteria. With the Advance search users can build very granular searches of arbitrary complexity. Users may specify specifics about the term being searched, such as should the exact term only be searched, or should the search consider "contained in", "begins with", and/or "ends with". Terms may also be ANDed (&) and ORed (|) together and grouped with parathesis.

Symbols

  • ^ - the match must start at the beginning of the string
  • $ - the match must occur at the end of the string
  • ~ - Word boundary - Starts and/or ends a word. The match must occur on a boundary between an alphanumeric and a nonalphanumeric (e.g., blank/whitespace) character.
  • "" or '' - wrap multiple text strings

  • & - AND operator
  • | - OR operator
  • () - grouping constructs delineating sub conditions of an expression, e.g., ((this | that) & ('something else'))
Examples

The table below shows some examples of advanced search criteria and what each would return.

^microservice Node begins with microservice microservice, microservices MyMicroservice, "Any text microservice", "Any text microservices"
^microservice$ Exact string microservice microservice ONLY microservices, MyMicroservice, "Any text microservice"
