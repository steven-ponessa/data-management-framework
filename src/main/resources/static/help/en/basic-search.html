Basic search comes in the form of filters for a multi-select box. Each option in the select box is a valid node within the Job Role/Specialty taxonomy.

The select box provides all the features of a standard select box with the addition of:

  • Grouping options by taxonomy level
    1. Brand
    2. Growth Platform
    3. Service Line
    4. Practice
    5. Service Area
    6. Job Role Specialty (JRS)
  • Type-ahead filtering. As the user types, options (taxonomy nodes) that do not contain the string entered are removed from the list of options.
  • Once an option has been selected, it is removed from the list of available options within the group it belongs.

As taxonomy nodes are selected or removed, they are searched for and found in the tree. The d3 tree is then re-drawn with branches of non-found nodes collapsed and found nodes expanded and the nodes, and arcs between the nodes, are highlighted in red.

The selection dropdown allows for multiple nodes to be selected. These selections are logically ORed together upon selection with a "pillbox" for the selection appearing within the search input box. These nodes/pillboxes may be removed/closed by clicking on the 'x' at the right of the pillbox. To request a search where nodes are ANDed together, use the Advanced search tab.

By default, the tree is searched ignoring case and using a pattern match. Therefore, if the node selected was SAP it would find any node containing SAP, regardless of case. To return only nodes that exactly match the node selected change the radio button selection to Exact match.
